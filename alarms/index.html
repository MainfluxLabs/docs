Alarms#

Alarms service persists triggered alarms to a PostgreSQL database. Alarms are generated by the Rules engine or Lua scripts when a message matches a defined condition.

Each alarm records: the thing that triggered it, the rule or script that caused it, the message subtopic and protocol, and the raw payload.

To start the Alarms service:

docker-compose -f docker/addons/alarms/docker-compose.yml up -d

For the full API reference, see the API documentation.
